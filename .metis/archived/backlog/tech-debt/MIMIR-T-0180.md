## Objective **[REQUIRED]**

Add validation to service filter structs to prevent unbounded queries (e.g., empty LIKE patterns, excessive limit values) that could cause performance issues.

### Technical Debt Impact **[CONDITIONAL: Tech Debt]**
- **Current Problems**: Filter structs (`ActionFilters`, `SpellFilters`, etc.) accept any values without validation. Could allow `LIKE "%"` patterns, unlimited result sets, or other slow queries.
- **Benefits of Fixing**: Predictable query performance, protection against accidental DoS, clearer API contracts.
- **Risk Assessment**: Low immediate risk in desktop app context, but bad patterns could cause issues as data grows or if exposed via API.
